# Milestone 4 Chaos Experiments

This folder contains the Chaos Mesh manifests used to satisfy the Milestone 4 requirements for:

1. a pod/service kill test
2. a network latency test

## Framework

The repo uses Minikube for the Kubernetes deployment path established in Milestone 3, so Chaos Mesh
is the selected chaos framework for Milestone 4.

## Target namespace

The application workloads are deployed into:

- `ab-calicritters`

The Chaos Mesh control plane is installed into:

- `chaos-mesh`

## Manifests

Pod kill:

- `events-api-pod-kill.yaml`
- `assignment-api-pod-kill.yaml`

Network latency:

- `api-network-latency.yaml`

The latency manifest was validated with Chaos Mesh using `direction: to`. An earlier `direction:
both` configuration was rejected by Chaos Mesh for this scenario because no explicit target set was
defined for bidirectional shaping.

## Suggested execution flow

1. Install Chaos Mesh:
   - `powershell -ExecutionPolicy Bypass -File scripts/minikube/chaos-install.ps1`
2. Reconfirm baseline deployment health:
   - `powershell -ExecutionPolicy Bypass -File scripts/minikube/smoke-test.ps1 -OutputDir artifacts/milestone4`
3. Run the kill experiment:
   - `powershell -ExecutionPolicy Bypass -File scripts/minikube/chaos-kill-test.ps1 -Target events-api`
   - `powershell -ExecutionPolicy Bypass -File scripts/minikube/chaos-kill-test.ps1 -Target assignment-api`
4. Run the latency experiment:
   - `powershell -ExecutionPolicy Bypass -File scripts/minikube/chaos-latency-test.ps1`
5. Clean up:
   - `powershell -ExecutionPolicy Bypass -File scripts/minikube/chaos-cleanup.ps1`

## Notes

1. The selectors in these manifests are aligned with the current Milestone 3 deployments.
2. If deployment labels change later, update the selectors here before running the experiments.
3. Evidence generated by the helper scripts should be written under `artifacts/milestone4/`.
4. The Milestone 4 runs confirmed a real resilience gap in single-replica `assignment-api`, so
   `infra/k8s/assignment-api-deployment.yaml` was updated to use two replicas before the re-test.
